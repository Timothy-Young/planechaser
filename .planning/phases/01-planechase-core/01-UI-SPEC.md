# Phase 1 — UI Design Contract: Planechase Core

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Style direction:** Retro-Futurism — dark fantasy gaming aesthetic. Vintage sci-fi, neon glow, geometric patterns. MTG's multiplanar universe rendered on an OLED dark canvas.

---

| Property | Value |

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline micro-spacing |
| sm | 8px | Compact element spacing, minimum gap between touch targets |
| md | 16px | Default element spacing, form field padding, card body padding |
| lg | 24px | Section padding, panel internal spacing |
| xl | 32px | Layout gaps between major regions |
| 2xl | 48px | Major section breaks, screen-level vertical rhythm |
| 3xl | 64px | Page-level spacing, safe-area bottom padding on mobile |

**Exceptions:**
- Roll Die button minimum height: 64px (hero action — supersedes standard xl)
- All interactive touch targets minimum: 44px height AND width (WCAG 2.1 AA)
- Chaos counter badge minimum tap area: 44×44px even when visually smaller
- Player list row minimum height: 48px
- Bottom safe-area clearance on iOS: add `env(safe-area-inset-bottom)` below Roll Die button

## Color

| Role | Hex | CSS Variable | Usage |
|------|-----|--------------|-------|
| Dominant (60%) | `#0F0F23` | `--color-bg` | App background, screen surfaces, OLED dark base |
| Secondary (30%) | `#1A1A3E` | `--color-surface` | Cards, panels, overlays, nav bar, bottom sheet |
| Accent Primary (10%) | `#7C3AED` | `--color-accent` | Active states, focus rings, current plane border glow, chaos counter ring |
| Accent Secondary | `#A78BFA` | `--color-accent-muted` | Secondary interactive elements, player list highlight, visited indicator |
| CTA | `#F43F5E` | `--color-cta` | Roll Die button background only |
| CTA Hover | `#E11D48` | `--color-cta-hover` | Roll Die button on hover/press |
| Text Primary | `#E2E8F0` | `--color-text` | All body text, headings |
| Text Muted | `#94A3B8` | `--color-text-muted` | Secondary text, captions, disabled states |
| Destructive | `#DC2626` | `--color-destructive` | "End Game" confirmation, abandon session button |
| Border | `#2D2D5E` | `--color-border` | Panel borders, dividers, form field outlines |

**Accent (`#7C3AED`) reserved for:**
1. Current plane card border glow (game screen)
2. Active focus rings (keyboard/pointer focus visible state)
3. Chaos cost counter ring/badge
4. Neon glow on "currently active" player indicator
5. Selected state on toggle controls (e.g. "exclude visited" toggle ON)

**CTA (`#F43F5E`) reserved for:**
1. Roll Die button — the single hero CTA — exclusively
2. Roll Die button in its disabled/rolling state: opacity 0.6, same color (no color change during animation)

**Do not use CTA color on any element other than Roll Die.**

**Global surface rules:**
- All backgrounds: OLED dark only (`#0F0F23` or `#1A1A3E`) — no white, no light surfaces
- All text must meet 4.5:1 contrast minimum against its background (`#E2E8F0` on `#0F0F23` = ~13:1 — passes)
- `#94A3B8` (muted text) on `#0F0F23` = ~5.2:1 — passes AA
- `#94A3B8` on `#1A1A3E` = verify at implementation; use `#E2E8F0` if contrast check fails

## Effects Contract

These effects are part of the visual identity. All must respect `prefers-reduced-motion`.

| Effect | Implementation | Applies To | Reduced Motion Fallback |
|--------|---------------|------------|------------------------|
| CRT scanlines | `::before` pseudo, `repeating-linear-gradient`, 2px lines, 0.05 opacity | App root wrapper | Disabled entirely |
| Neon glow | `box-shadow: 0 0 8px 2px #7C3AED80` + `text-shadow: 0 0 12px #7C3AED` | Accent-designated elements, Roll Die button (use `#F43F5E`) | Remove shadow, keep color |
| Card entry | Framer Motion `AnimatePresence` — new card slides in from bottom (y: 40→0) with spring physics (stiffness: 200, damping: 25) while old card fades out (opacity: 1→0, scale: 0.95) | Plane card on Planeswalk | Instant swap, no animation |
| Die glitch | 3-frame CSS keyframe glitch (translate ±3px, opacity flicker) for 0.3s on die result reveal | Die Result Overlay — result text only | Skip glitch, show result immediately |
| Plane card shimmer | Tailwind `animate-pulse` on a `#1A1A3E` skeleton | Plane card while Scryfall image loads | Same (shimmer is subtle, acceptable) |
| Focus ring | `outline: 2px solid #7C3AED; outline-offset: 2px` | All focusable elements | Same (focus visibility is accessibility, not decorative) |

## Screen-by-Screen Design Contracts

### Screen 1: Auth Screen

**Layout:** Single-column centered, max-width 400px, vertically centered on viewport.

**Elements (top to bottom):**
1. App logo / wordmark — "PlaneChaser" in Russo One 28px, accent glow (`#7C3AED`)
2. Tagline — "The multiplanar conquest companion" in Chakra Petch 14px muted text
3. Form card (surface `#1A1A3E`, border `#2D2D5E`, border-radius 12px, padding 24px)
   - Email input — Chakra Petch 16px, height 48px, border `#2D2D5E`, focus border `#7C3AED`
   - Password input — same spec as email
   - "Sign In" button — full width, height 48px, background `#7C3AED`, Russo One 16px, white text
   - OR divider — 1px `#2D2D5E` with "OR" label in Chakra Petch 12px muted
   - "Continue with Google" button — full width, height 48px, surface `#2D2D5E`, Chakra Petch 14px 600, Google icon (Lucide or SVG), white text
4. "Create account" toggle — Chakra Petch 14px, accent color link below form card
5. WotC disclaimer (first-launch only — AUTH-05) — bottom of screen, Chakra Petch 12px muted, auto-dismissed after 3 seconds or tap

**WotC disclaimer copy:**

**States:** idle → loading (button spinner, opacity 0.7) → error (inline below input, Chakra Petch 14px `#F43F5E`) → success (redirect)

### Screen 2: Game Setup Screen

**Layout:** Single-column, full-width, 16px horizontal padding. Header + scrollable form body.

**Elements (top to bottom):**
1. Header: "New Game" in Russo One 20px. Back arrow (Lucide `ChevronLeft`, 24px, 44×44px tap area) top-left.
2. Form sections (each in a surface card `#1A1A3E`, 16px padding, 12px border-radius, 8px gap between cards):
   - **Players** — Chakra Petch 14px 600 label. Horizontal segmented control: "2" "3" "4" "5" "6". Each segment 44px tall, flex-grow equal. Active: accent fill `#7C3AED`. Inactive: surface `#2D2D5E`.
   - **Deck Size** — Chakra Petch 14px 600 label + current value (e.g. "40 cards") right-aligned. Slider input, full width. Track: `#2D2D5E`. Thumb: `#7C3AED`, 24×24px. Range: 10–86 (full Planechase corpus). Default: 40.

### Screen 3: Game Screen (Main)

**Layout:** Vertical flex, full viewport height. No scroll on the main view — all content fits. Card art is the hero visual.

**Zone breakdown (375px wide, ~812px tall reference):**
```
┌─────────────────────────────┐
│  Header bar (48px)          │  Player count badge · "End Game" button
├─────────────────────────────┤
│                             │
│   Plane Card (hero)         │  ~340px tall · full width · 16px horizontal margin
│   Scryfall art fills area   │  border-radius 12px · accent glow border 1px
│                             │
│   Plane name (over card)    │  Russo One 28px · white · bottom of card · text-shadow
│   Set name                  │  Chakra Petch 12px muted
├─────────────────────────────┤
│  Chaos Counter (56px)       │  Always-visible counter strip
├─────────────────────────────┤
│  Player List (variable)     │  Compact rows · 48px each · max 4 visible · scrollable if 5+
├─────────────────────────────┤
│  Roll Die button (80px)     │  64px button height + 8px top/bottom padding
│  + safe-area inset          │  Full width · 16px margin
└─────────────────────────────┘
```

**Plane Card:**
- `next/image` component, `src` from `cards.scryfall.io` (normal size: 488×680px)
- Aspect ratio: locked to Scryfall card proportions (approximately 5:7)
- Loading state: `#1A1A3E` skeleton with Tailwind `animate-pulse`
- Error state: dark card with plane name in Russo One, Lucide `ImageOff` icon center, Chakra Petch 12px muted "Art unavailable"
- Border: 1px `#7C3AED` with `box-shadow: 0 0 8px 2px #7C3AED40` (accent-designated)
- `alt` text: `"{Plane Name} — {set name} plane card art"` (GAME-01 accessibility)

**Chaos Counter:**
- Strip background: `#1A1A3E`, full width, 56px height, 16px padding
- Label: "Chaos Cost" Chakra Petch 14px 600 muted, left-aligned
- Counter: current value as `{N}{1}` e.g. "0{1}" or "2{1}" — Russo One 20px `#E2E8F0`, right-aligned
- Ring badge around counter: 2px `#7C3AED` circle (accent-designated)
- Tooltip/tap: none — the counter must be visually clear without interaction (GAME-03 requirement)

**Player List:**
- Each row: 48px height, `#1A1A3E` background, 16px horizontal padding
- Player name: Chakra Petch 14px 600 `#E2E8F0`
- Active turn indicator (if tracked): 4px left border `#7C3AED` (accent-designated)
- Dividers: 1px `#2D2D5E`

**Roll Die Button:**
- Height: 64px minimum (SUMMARY.md pitfall #4)
- Width: full width minus 32px (16px each side)
- Background: `#F43F5E` (CTA color — exclusively this element)
- Label: "Roll Die" — Russo One 20px white
- Box shadow: `0 0 16px 4px #F43F5E40` (neon glow with CTA color)
- ROLLING state: disabled, opacity 0.6, label replaced by Lucide `Loader2` spinning icon + "Rolling..." text
- Tap area: minimum 64px height already satisfies 44px requirement

### Screen 4: Die Roll Result Overlay

**Trigger:** Roll Die tapped → state machine: IDLE → ROLLING → RESOLVING (this overlay) → IDLE

**Layout:** shadcn `Dialog` or full-screen overlay. Background: `rgba(15, 15, 35, 0.92)` blur backdrop. Center-aligned content panel (surface `#1A1A3E`, 280px wide max, border-radius 16px, 32px padding).

**Content by result type:**

**Planeswalk:**
- Result label: "PLANESWALK" — Russo One 28px `#A78BFA` (accent muted) with 0.3s glitch effect on entry
- Sub-label: "A new plane awaits..." — Chakra Petch 14px muted
- Dismiss: auto-dismiss after 1.5s AND tappable anywhere to skip
- On dismiss: trigger card transition animation (old plane exits, new plane enters via Framer Motion AnimatePresence)

**Chaos:**
- Result label: "CHAOS" — Russo One 28px `#F43F5E` (CTA color reused semantically for chaos danger) with glitch effect
- Chaos ability text: full oracle text from Scryfall, Chakra Petch 16px `#E2E8F0`, displayed in scroll region if long (max 200px, overflow-y auto)
- Highlight: chaos text background `#2D2D5E`, 8px padding, 8px border-radius
- Sub-label: "Resolve the chaos ability, then continue." — Chakra Petch 14px muted
- Dismiss: tap anywhere OR "Got It" button (Chakra Petch 14px 600, `#7C3AED`, height 44px, full width)

**Blank:**
- Result label: "BLANK" — Russo One 28px `#94A3B8` (muted — blank is uneventful)
- Sub-label: "Nothing happens. Pay {1} to roll again." — Chakra Petch 14px muted
- Chaos cost line: "Current chaos cost: {N}{1}" — Chakra Petch 14px 600 `#E2E8F0`
- Dismiss: tap anywhere — auto-dismisses after 2s

**Chaos cost escalation visibility:**
- After any roll (not just Blank): chaos counter on Game Screen updates immediately when overlay dismisses

## State Machine: Roll Die

The die button is the most-used interaction. It must be mechanically airtight.

```
IDLE
  └─ tap Roll Die ──────────────────────► ROLLING
                                             │
                                         (400ms animation)
                                             │
                                             ▼
                                         RESOLVING
                                        /    |    \
                                  Planeswalk Chaos  Blank
                                       │      │      │
                                   (overlay)(overlay)(overlay)
                                       │      │      │
                                   dismiss dismiss dismiss
                                       │      │      │
                                       └──────┴──────┘
                                                │
                                              IDLE
```

- ROLLING: Roll Die button disabled (`pointer-events: none`). No double-tap possible.
- RESOLVING: Die Roll Result Overlay visible. Roll Die button still disabled.
- Re-entry to IDLE only after overlay fully dismissed.
- Chaos cost counter increments on every roll that is NOT the first roll of a turn (GAME-03 logic — implemented in game engine, counter displayed from state).
