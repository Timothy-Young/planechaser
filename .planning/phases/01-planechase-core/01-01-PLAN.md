<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js middleware | Session token in HttpOnly cookie crosses this boundary on every request |
| Next.js server → Supabase | Service role and anon key; anon key is public-facing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | src/middleware.ts | mitigate | Use `getUser()` not `getSession()` — enforces server-side token revalidation against Supabase Auth on every request |
| T-01-02 | Tampering | supabase/migrations/001_initial_schema.sql | mitigate | profiles.display_name has UNIQUE constraint — duplicate username registration blocked at DB level |
| T-01-03 | Elevation | src/middleware.ts route protection | mitigate | Middleware redirects unauthenticated requests to /auth for all /game and /setup paths; enforced server-side |
| T-01-04 | Information Disclosure | .env.local.example | accept | NEXT_PUBLIC_* vars are intentionally public (Supabase anon key is meant to be client-visible); RLS enforces data access control |
</threat_model>
