# Quick Task 260702-xa8: Pre-launch Supabase security hardening

**Created:** 2026-07-03
**Mode:** quick (executed inline ŌĆö DB work requires Supabase MCP access unavailable to subagents)

## Objective

Close the security gaps found by Supabase advisors before making PlaneChaser public:
remove foreign tiktok-dashboard tables, and fix RLS/function/view/storage findings.

## Investigation findings (locked decisions)

1. **TikTok tables** (`tiktok_accounts`, `videos`, `scripts`, `products`, `briefings`, `pipeline_runs`, `competitor_videos`): zero references in PlaneChaser code, **all 0 rows** ŌĆö drop without backup. They belong to a separate tiktok-dashboard project; Tim confirmed keeping the projects separate.
2. **`admin_user_stats` view**: SECURITY DEFINER (advisor ERROR). All underlying tables (`profiles`, `game_sessions`, `conquered_planes`, `custom_planes`, `feedback`) have admin read-all RLS policies via `get_my_role()`, so switching to `security_invoker=true` keeps the admin dashboard working.
3. **`conquered_planes`**: INSERT `WITH CHECK (true)` + two duplicate DELETE `USING (true)` policies. App inserts rows for OTHER users (host records winner's conquest; dethrone inserts for new owner ŌĆö `planechaser/src/lib/pods/queries.ts`), so policies must scope to **pod membership**, not `auth.uid() = user_id`. `pod_id`/`user_id` are NOT NULL; `pod_members` SELECT is `true` for authenticated ŌåÆ plain EXISTS subqueries are safe (no recursion).
4. **`card_cache`**: INSERT/UPDATE open to all authenticated users, but no live source references it (only old research docs) ŌĆö drop the write policies, keep table + SELECT.
5. **SECURITY DEFINER functions**: `get_my_role` and `get_user_session_ids` are used inside RLS policies ŌåÆ keep EXECUTE for `authenticated`, revoke `anon`/`PUBLIC`. Trigger functions (`handle_new_user`, `protect_role_changes`, `set_feedback_email`) need no caller EXECUTE ŌåÆ revoke from all API roles. Pin `search_path` on the four functions missing it (use `public`, not `''` ŌĆö `protect_role_changes` calls `get_my_role()` unqualified).
6. **Storage**: app only uses `getPublicUrl`/`upload`/`remove` on `custom-plane-images` (never `.list()`) ŌåÆ drop the broad SELECT policy; public buckets serve objects without it, scoped INSERT/UPDATE/DELETE policies remain.
7. **Leaked-password protection**: dashboard-only toggle, not automatable via MCP ŌĆö surface as manual step for Tim.
